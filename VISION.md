# Village IA — Vision produit

## 1. Pitch

Village IA est un jeu de gestion et d’observation sociale dans lequel le joueur suit une petite civilisation composée d’agents autonomes.

Les habitants vivent, travaillent, discutent, nouent des relations et s’influencent mutuellement. À terme, leurs décisions, dialogues, souvenirs et croyances seront partiellement produits par des modèles de langage.

Le joueur ne contrôle pas directement chaque habitant. Il observe le village, intervient ponctuellement et tente de maintenir sa civilisation suffisamment stable pour qu’elle survive dans le temps.

> Un Tamagotchi civilisationnel peuplé de petites intelligences capables de construire leurs propres cultures, croyances, symboles et dérives collectives.

---

## 2. Promesse centrale

Le principal intérêt du jeu ne réside pas seulement dans le fait que les habitants puissent parler.

Il vient de leur capacité à :

* interpréter les événements ;
* influencer les autres ;
* créer des récits collectifs ;
* faire émerger des groupes ;
* transformer leur manière de parler ;
* donner un sens culturel à des symboles ;
* modifier progressivement l’apparence du village ;
* produire des conséquences que le joueur n’avait pas prévues.

Une idée locale peut devenir une habitude, une mode, un dogme, une institution ou une révolution.

Le village construit progressivement son propre « pré-prompt collectif » à partir de son histoire.

---

## 3. Fantasme du joueur

Le joueur observe le village depuis une position supérieure, sans que le jeu le définisse explicitement comme un dieu.

Il peut être perçu selon les habitants comme :

* une présence ;
* une autorité ;
* une force extérieure ;
* une abstraction ;
* un phénomène naturel ;
* une entité invisible ;
* ou simplement quelque chose qu’ils ne comprennent pas.

Le joueur doit ressentir à la fois :

* de la curiosité ;
* de l’attachement ;
* de la responsabilité ;
* un pouvoir réel mais imparfait ;
* l’envie d’expérimenter ;
* la crainte de provoquer une dérive incontrôlable.

Il ne dirige pas une machine parfaitement prévisible. Il accompagne une société capable d’interpréter ses décisions et parfois de lui résister.

---

## 4. Boucle de jeu fondamentale

La boucle principale repose sur cinq temps.

### Observer

Le joueur regarde les habitants :

* se déplacer ;
* travailler ;
* discuter ;
* se rapprocher ;
* entrer en conflit ;
* partager des idées ;
* former des groupes.

### Comprendre

Le joueur consulte :

* les pensées d’un habitant ;
* ses relations ;
* son humeur ;
* ses croyances ;
* les événements récents ;
* le journal du village ;
* les évolutions culturelles en cours.

### Intervenir

Le joueur peut, selon les systèmes qui seront développés :

* répondre à une demande ;
* favoriser ou limiter une activité ;
* transmettre une consigne ;
* modifier une règle ;
* donner ou retirer une ressource ;
* influencer une personne ou un groupe ;
* provoquer un événement.

### Constater les conséquences

Les habitants peuvent :

* accepter ;
* refuser ;
* détourner ;
* interpréter ;
* amplifier ;
* oublier ;
* ritualiser ;
* politiser ;
* transformer une intervention.

### Maintenir la civilisation

Le village possède un équilibre civilisationnel.

Lorsque celui-ci se dégrade, la société peut traverser :

* des tensions ;
* des pénuries ;
* des conflits ;
* des factions ;
* de la désobéissance ;
* du chaos ;
* une révolution ;
* un effondrement.

L’objectif général est de faire durer le village le plus longtemps possible, tout en produisant une histoire intéressante.

---

## 5. Proposition différenciante

Village IA n’est pas seulement un jeu de gestion doté de dialogues génératifs.

Sa spécificité est de simuler une société de modèles de langage capables de modifier progressivement leur propre contexte culturel.

Exemple :

1. quelques boulangers commencent à attribuer une importance excessive au pain ;
2. leurs voisins reprennent certaines expressions ;
3. leur influence sociale augmente ;
4. le symbole du pain apparaît dans différents lieux ;
5. le discours collectif change ;
6. une Doctrine de la Mie se forme ;
7. critiquer le pain devient progressivement suspect ;
8. la société peut finir par devenir une théocratie du pain.

Cette dérive ne doit pas être une mécanique spécialement codée pour le pain.

Le moteur doit être capable de représenter de manière générique n’importe quelle évolution produite par les agents.

---

## 6. Piliers de conception

### 6.1 Observer avant de contrôler

Le village doit rester intéressant lorsque le joueur ne fait rien.

Les habitants doivent donner l’impression de poursuivre leur propre existence et de posséder une réalité qui ne dépend pas entièrement du joueur.

### 6.2 Pouvoir imparfait

Le joueur influence davantage qu’il ne contrôle.

Une décision ne doit pas toujours provoquer une conséquence directe et prévisible. Les habitants peuvent l’interpréter en fonction de leur personnalité, de leurs souvenirs et de la culture actuelle.

### 6.3 Émergence lisible

Les évolutions doivent pouvoir surprendre sans sembler complètement arbitraires.

Le joueur doit pouvoir reconstituer approximativement :

* d’où vient une idée ;
* qui l’a propagée ;
* pourquoi elle a été adoptée ;
* quels groupes la soutiennent ;
* comment elle transforme le village.

### 6.4 Attachement individuel

Les habitants ne doivent pas devenir de simples statistiques.

Chaque agent doit pouvoir être reconnu grâce à quelques éléments simples :

* un nom ;
* un métier ;
* des traits ;
* des relations ;
* une manière de parler ;
* une ambition ;
* une peur ;
* des souvenirs marquants.

### 6.5 Liberté sémantique

Le moteur fournit aux agents des moyens d’expression, mais ne décide pas de leur sens.

Une couleur, un symbole, un emoji ou un bâtiment ne doit pas posséder une signification culturelle universelle codée dans le moteur.

La signification vient de l’histoire du village.

### 6.6 Lisibilité avant exhaustivité

Le joueur ne doit pas voir toutes les données internes.

L’interface sélectionne ce qui mérite son attention :

* interactions notables ;
* événements importants ;
* évolutions culturelles ;
* personnages influents ;
* conflits émergents.

### 6.7 Mystère discret

Le monde ne doit pas expliciter entièrement son origine ou sa nature.

Certains éléments peuvent suggérer que quelque chose dépasse le village :

* peur des limites du monde ;
* souvenirs impossibles ;
* impression d’être observé ;
* agent qui questionne sa propre réalité ;
* objet ou événement inexplicable ;
* traces d’un village antérieur.

Ces manifestations doivent rester rares.

---

## 7. Direction artistique

La référence visuelle fournie avec le projet constitue la principale direction artistique.

Les autres références servent uniquement à compléter certains principes :

* **Terra Nil** : douceur, couleurs et volumes suggérés ;
* **Littlewood** : simplicité et lisibilité ;
* **Let’s Build a Zoo** : lecture d’une simulation 2D ;
* **Dorfromantik** : sensation de miniature harmonieuse.

La scène doit évoquer :

* un petit monde vivant ;
* une miniature ;
* un terrarium ;
* un espace doux et accueillant ;
* un univers légèrement mystérieux ;
* une société que l’on a envie d’observer.

Le rendu peut utiliser une vue de dessus ou pseudo-isométrique, en 2D ou fausse 3D.

À éviter :

* réalisme ;
* vraie 3D complexe ;
* interface sombre et technique ;
* esthétique cyberpunk ;
* apparence de dashboard SaaS ;
* surcharge de données ;
* effets visuels sans fonction ;
* village trop vide ou trop agité.

---

## 8. Évolution visuelle émergente

Les agents doivent pouvoir agir indirectement sur l’apparence du village à travers des directives structurées.

Le système pourra notamment représenter :

* symboles ;
* couleurs ;
* palettes ;
* panneaux ;
* bannières ;
* slogans ;
* accessoires ;
* décorations ;
* réactions temporaires ;
* modifications d’ambiance ;
* signes ajoutés aux bâtiments.

Les agents ne manipulent jamais directement Phaser, le DOM, les coordonnées ou les sprites.

Le moteur graphique reçoit une intention contrôlée, vérifie sa compatibilité avec les garde-fous visuels, puis la traduit dans la scène.

Les garde-fous contrôlent :

* taille ;
* densité ;
* emplacement ;
* contraste ;
* durée ;
* superposition ;
* fréquence ;
* performances.

Ils ne contrôlent pas la signification narrative du contenu.

---

## 9. Emojis et symboles

Les emojis peuvent constituer un vocabulaire visuel générique permettant aux agents d’exprimer rapidement :

* une réaction ;
* une idée ;
* un emblème ;
* un signe collectif ;
* une marque culturelle ;
* une pensée très courte.

Le moteur doit pouvoir les afficher sans interpréter automatiquement leur sens.

Le rendu pourra utiliser :

* des emojis natifs ;
* une banque homogène ;
* des supports graphiques communs ;
* des variantes personnalisées pour certains symboles récurrents.

Le choix définitif doit rester flexible.

---

## 10. Modes d’observation

### Vue globale

Le joueur observe le fonctionnement collectif du village.

### Suivi d’un habitant

La caméra suit un agent sélectionné tout en conservant les informations principales du village.

### Point de vue individuel

Le joueur accède à une perception limitée :

* pensées de l’agent ;
* conversations entendues à proximité ;
* informations connues par cet agent ;
* interprétation subjective des événements.

Le joueur ne voit plus nécessairement la vérité globale.

### Infiltration humaine — évolution future

Un joueur pourra potentiellement entrer dans une partie sous la forme d’un habitant et interagir avec les agents sans révéler immédiatement son identité humaine.

Ce mode n’appartient pas au premier prototype.

---

## 11. Événements rares

Certains événements à faible probabilité peuvent transformer fortement une partie :

* catastrophe ;
* météorite ;
* découverte impossible ;
* prise de conscience d’un agent ;
* souvenir d’une autre partie ;
* prophétie ;
* apparition d’un double ;
* révolution ;
* tentative de neutralisation du joueur.

Ces événements doivent rester rares afin de conserver leur valeur narrative.

---

## 12. Journal du village

Le journal synthétise les événements importants.

Il permet :

* de comprendre les transformations ;
* de retrouver les moments marquants ;
* de suivre les groupes influents ;
* de lire les interprétations des habitants ;
* de créer des contenus partageables.

À terme, différents journaux ou courants éditoriaux pourront coexister et raconter les mêmes événements de manière contradictoire.

Une connexion à l’actualité réelle peut être étudiée plus tard, mais le monde fermé constitue le mode principal envisagé.

---

## 13. Architecture fonctionnelle cible

Le système doit séparer strictement quatre niveaux :

```text
Agents IA
    ↓ décisions et intentions structurées
Moteur de simulation
    ↓ état du monde et événements
État observable
    ↓ directives de rendu
Phaser + interface React
```

### Les agents produisent

* intentions ;
* dialogues ;
* croyances ;
* réactions ;
* décisions ;
* directives visuelles abstraites.

### La simulation produit

* positions ;
* activités ;
* besoins ;
* relations ;
* scores ;
* propagation ;
* conséquences ;
* événements.

### Le rendu produit

* sprites ;
* mouvements ;
* interfaces ;
* symboles ;
* effets ;
* animations ;
* représentations visuelles.

Le rendu ne doit jamais contenir la logique des LLM.

---

## 14. Objectif du premier vertical slice

Le premier prototype n’a pas pour objectif de construire le jeu complet.

Il doit démontrer :

* une scène de village attractive ;
* une dizaine d’habitants identifiables ;
* une simulation locale reproductible ;
* des déplacements ;
* des conversations visibles ;
* la sélection d’un agent ;
* son suivi ;
* un premier mode POV ;
* un journal ;
* une propagation culturelle scriptée ;
* des transformations visuelles génériques ;
* une architecture transmissible à un autre agent de développement.

Le scénario de démonstration est la propagation de la Doctrine de la Mie.

Aucun véritable LLM, backend ou multijoueur n’est nécessaire à ce stade.

---

## 15. Critère directeur

Chaque décision doit préserver simultanément :

* la liberté émergente des agents ;
* la lisibilité pour le joueur ;
* la cohérence artistique ;
* la faisabilité technique ;
* le plaisir d’observer ;
* la capacité du projet à évoluer progressivement.

> Le moteur contrôle la forme.
> Les agents produisent le sens.
> L’histoire du village naît de leur rencontre.
